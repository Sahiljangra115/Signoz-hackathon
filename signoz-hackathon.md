## Blog Guide (for pre-event blog prize)

### Core rule
Write from **real experience**, not summarized docs/research.

### Strong post signs
- Solves real problem or clarifies confusing concept
- Insider details only a practitioner would know
- Reader can replicate results
- Uses SigNoz's full feature set
- Has real feedback + conclusions

### Weak post signs (avoid)
- Mirrors official docs
- Vague generalizations
- Reads like author never touched terminal
- AI filler with no substance

### Content angles (pick one)
1. **What I built** — walkthrough: problem, solution, outcome, lessons
2. **How to do X** — step-by-step tutorial
3. **What I learned** — concepts, gotchas, debugging stories
4. **Why / my take** — opinion backed by reasoning + examples

Rule: "narrow beats broad" — specific > generic.

### Structure

1. **Hook** (2-3 sentences) — problem/surprising fact/question, state reader benefit
2. **Context** — what and why, concise
3. **Main body** — steps, code, config, reasoning
4. **Learnings/Takeaways** — what worked, what failed, retrospective
5. **Conclusion** — one-line summary + resources

Target length: **1,000-1,500 words**

### Technical requirements
- Real code/config examples
- Screenshots (dashboards, traces, outputs) — use sparingly
- Personally test all commands/steps
- Verify technical claims against official sources (e.g. opentelemetry.io)
- Acknowledge scope limits ("worked for my setup")

### Avoid
- LinkedIn post reframed as blog
- Unpublished/proprietary platforms
- AI-generated text w/o real experience
- Plagiarism (quote + link instead)
- Sales-y framing over usefulness
- Burying key insight — front-load value

### Submission checklist
- [ ] All commands/steps tested
- [ ] Technical claims verified against official sources
- [ ] Relevant SigNoz screenshots included (not flooded)
- [ ] Published before July 19, 2026 deadline
- [ ] Posted on Medium/Dev.to/Substack (not LinkedIn)
